# Plan: Arreglar carga y creación de usuarios con rol Instructor

## Problemas identificados

### 1. Modelo Rol (`src/app/domain/models/rol.model.ts`)
El frontend usa nombres de propiedades diferentes al backend:
| Frontend | Backend |
|----------|---------|
| `id` | `id_rol` |
| `nombreRol` | `nombre` |

### 2. Modelo Usuario (`usuario.model.ts`)
El frontend NO tiene los campos que requiere el backend:
| Backend requiere | Frontend tiene (o falta) |
|------------------|-------------------------|
| `nombre` | ❌ `nombreCompleto` (diferente) |
| `id_rol` | ❌ `rolId` (diferente nombre) |
| `password` | ❌ NO existe en modelo (requerido para crear) |
| `apellidos` | Frontend envía pero backend NO lo espera |
| `telefono` | Frontend envía pero backend NO lo espera |
| `documento` | Frontend envía pero backend NO lo espera |

## Solución

### 1. Actualizar `src/app/domain/models/rol.model.ts`
```typescript
export interface Rol { 
  id_rol: number; 
  nombre: string; 
  estado?: string | boolean;
}
```

### 2. Actualizar `src/app/domain/models/usuario.model.ts`
```typescript
export interface Usuario { 
  id_usuario?: number;
  nombre?: string;
  correo: string;
  telefono?: string;
  documento?: string;
  password?: string;
  estado: boolean;
  id_rol: number;
  rolNombre?: string;
}
```

### 3. Verificar `usuarios.component.ts`
- `optionValue="id_rol"` en el `<p-select>` (ya corregido)
- El método `cargarDatos` ya maneja correctamente el mapeo (líneas 284-297)

### 4. Verificar que el backend acepte los campos
El backend espera `nombre`, `correo`, `password`, `id_rol`. Los campos `apellidos`, `telefono`, `documento` no están en el DTO del backend.
